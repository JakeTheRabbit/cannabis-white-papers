# Cannabis White Papers - remote MCP server

A stateless Streamable-HTTP [MCP](https://modelcontextprotocol.io) server on a
Cloudflare Worker, backed by Workers KV. Tools: `search_papers`, `get_paper`,
`list_papers`, `search_glossary`, `get_glossary_term`.

The corpus (`data/`) is generated by `_build/export_corpus.py` (run `python build.py`).

## Deploy

```bash
cd mcp
npm install

# auth: either `wrangler login`, or set CLOUDFLARE_API_TOKEN (+ CLOUDFLARE_ACCOUNT_ID)
# with Workers Scripts:Edit and Workers KV Storage:Edit scopes.

# 1. create the KV namespace, paste the returned id into wrangler.toml
wrangler kv namespace create CORPUS

# 2. load the corpus into KV (data/kv-bulk.json is produced by the exporter)
wrangler kv bulk put data/kv-bulk.json --binding CORPUS --remote

# 3. deploy
wrangler deploy
```

## Use

Add the deployed URL as a custom connector in Claude (or any MCP client):

```
https://cwp-mcp.<your-subdomain>.workers.dev/mcp
```

Read-only, public, no auth. Re-run steps 2 (kv bulk put) after each corpus update.
